# 融合时序特征的投融资风险知识图谱可视化系统

## SRT大学生研究训练计划研究报告

---

## 摘要

本项目基于张华《基于时序知识图谱的投资决策研究与实现》论文，设计并实现了一个融合时序特征的投融资风险知识图谱可视化系统（InvGraph）。系统采用四层架构设计，集成自然语言处理、知识图谱构建、时序推理等技术，实现投融资事件的自动抽取、关系建模和可视化分析。实验结果表明，系统在实体识别、关系抽取、时序分析等任务上达到了较好的效果，为投资决策提供了有效的数据支撑。

**关键词：** 知识图谱；时序推理；投融资分析；自然语言处理；可视化

---

## 1. 研究背景与意义

### 1.1 研究背景

随着金融市场的快速发展，投融资信息呈现爆发式增长。传统的投融资分析主要依赖人工收集和整理数据，效率低下且难以发现复杂的关联关系。知识图谱作为一种结构化的知识表示方法，能够有效整合多源异构数据，揭示实体间的复杂关系。

时序特征在投融资分析中具有重要意义。投资事件的时间顺序、投资轮次的演进、企业发展的阶段性特征等，都是投资决策的关键因素。然而，现有的知识图谱系统大多忽略了时序信息，难以支持动态的投融资关系分析。

### 1.2 研究意义

本项目的研究意义主要体现在以下几个方面：

1. **理论意义：** 探索时序知识图谱在投融资领域的应用，丰富知识图谱的研究内容
2. **实践意义：** 构建可实际应用的投融资分析系统，为投资决策提供数据支撑
3. **技术意义：** 整合多种NLP技术，实现投融资信息的自动化抽取和分析

### 1.3 研究目标

本项目的研究目标是设计并实现一个融合时序特征的投融资风险知识图谱可视化系统，具体包括：

1. 实现投融资文本的自动实体识别和关系抽取
2. 构建带时序属性的投融资知识图谱
3. 实现多维度的投融资关系可视化分析
4. 支持投资风险传导路径分析

## 2. 技术方案

### 2.1 系统架构

系统采用四层架构设计：

```
┌─────────────────────────────────────────────────────────────┐
│                      前端展示层                              │
│  Vue 3 + vis.js + Element Plus                              │
│  图谱可视化 | 时序分析 | 风险传导 | 投资热度                  │
├─────────────────────────────────────────────────────────────┤
│                      后端服务层                              │
│  Spring Boot 3.2 + MyBatis                                  │
│  API网关 | 数据查询 | 业务逻辑                               │
├─────────────────────────────────────────────────────────────┤
│                      ML服务层                               │
│  Python FastAPI + HanLP + PyTorch                           │
│  NER | 关系抽取 | 时序推理 | 实体对齐                        │
├─────────────────────────────────────────────────────────────┤
│                      数据存储层                              │
│  Neo4j 5.x | MySQL 8.0                                      │
│  图数据库 | 关系数据库                                       │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 技术栈选型

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| 前端 | Vue 3 + vis.js | 响应式框架+专业图谱可视化 |
| 后端 | Spring Boot | 成熟的企业级框架，生态完善 |
| ML服务 | Python FastAPI | 高性能异步框架，适合ML服务 |
| NLP | HanLP + BERT | 中文NLP能力强大 |
| 图数据库 | Neo4j | 专业图数据库，查询高效 |
| 关系数据库 | MySQL | 稳定可靠，易于维护 |

### 2.3 核心算法

#### 2.3.1 实体识别（NER）

系统采用HanLP预训练模型结合领域关键词的混合策略：

1. **HanLP NER：** 识别通用实体（人名、机构名）
2. **关键词匹配：** 识别投融资领域专有实体（轮次、行业）
3. **规则后处理：** 过滤噪声，标准化实体名称

#### 2.3.2 关系抽取

系统采用多模型融合策略：

1. **PL-Marker模型：** 基于BERT的实体标记关系抽取（权重0.6）
2. **SPN模型：** Set Prediction Network关系抽取（权重0.4）
3. **Stacking融合：** 加权投票提高准确率

#### 2.3.3 时序推理

系统实现两种时序推理方法：

1. **规则推理：** 基于投资轮次顺序和时间比较
2. **MTTR神经推理：** 多任务时序关系分类模型

## 3. 系统实现

### 3.1 数据采集与处理

#### 3.1.1 数据来源

系统从公开数据源采集投融资事件数据，包括：
- 投融资新闻报道
- 企业公告
- 行业研究报告

#### 3.1.2 数据清洗

数据清洗流程包括：
1. 去重处理（按企业+投资方+轮次+日期去重）
2. 格式标准化（日期格式、金额单位统一）
3. 实体规范化（公司全称、简称映射）

#### 3.1.3 数据规模

| 数据类型 | 数量 |
|----------|------|
| 企业 | 388家 |
| 投资机构 | 87家 |
| 行业 | 14个 |
| 投融资事件 | 4388条 |

### 3.2 NLP服务实现

#### 3.2.1 实体识别模块

```python
# 核心识别流程
def recognize(text):
    # 1. HanLP NER识别
    entities = hanlp_ner(text)

    # 2. 关键词补充
    entities += keyword_match(text)

    # 3. 规则后处理
    entities = post_process(entities)

    return entities
```

支持的实体类型：
- **Enterprise（企业）：** 支持公司全称、简称识别
- **Investor（投资方）：** 识别投资机构
- **Round（轮次）：** 识别融资轮次（天使轮、A轮、B轮等）
- **Industry（行业）：** 识别所属行业
- **Money（金额）：** 识别融资金额
- **Time（时间）：** 识别时间表达式

#### 3.2.2 关系抽取模块

支持的关系类型：
- **INVEST（投资）：** 一般投资关系
- **LEAD（领投）：** 领投关系
- **FOLLOW（跟投）：** 跟投关系
- **ACQUIRE（收购）：** 收购关系

#### 3.2.3 时序推理模块

时序关系类型：
- **BEFORE（之前）：** 事件A发生在事件B之前
- **AFTER（之后）：** 事件A发生在事件B之后
- **OVERLAP（重叠）：** 事件A和事件B时间重叠
- **SIMULTANEOUS（同时）：** 事件A和事件B同时发生

### 3.3 知识图谱构建

#### 3.3.1 图谱模型

```cypher
// 节点类型
(Enterprise)      // 企业节点
(Investor)        // 投资方节点
(Industry)        // 行业节点
(Round)           // 轮次节点

// 关系类型
[:INVEST {time, amount, round}]  // 投资关系（带时序属性）
[:BELONGS_TO]                     // 归属关系
[:LEAD]                           // 领投关系
[:FOLLOW]                         // 跟投关系
[:ACQUIRE]                        // 收购关系
```

#### 3.3.2 数据导入

支持两种导入方式：
1. **批量导入：** 从MySQL导入历史数据
2. **实时导入：** 从文本抽取结果直接导入

### 3.4 可视化实现

#### 3.4.1 图谱可视化

采用vis.js力导向图布局，支持：
- 节点按类型着色（企业-紫色、投资方-绿色、行业-橙色）
- 关系边显示投资金额和时间
- 支持缩放、拖拽、聚焦
- 点击节点查看详情

#### 3.4.2 时序可视化

- **时间轴视图：** 按时间顺序展示投资事件
- **热力图视图：** 展示行业投资热度分布
- **企业时间线：** 展示单个企业的融资历程

#### 3.4.3 风险分析可视化

- **风险传导路径：** 展示风险在投资网络中的传播路径
- **关联风险：** 识别存在关联风险的企业群

## 5. 核心创新点

### 5.1 时序四元组模型

创新性地提出时序四元组（主体-关系-客体-时间）模型，将时间信息融入知识图谱，支持：
- 按时间维度分析投融资关系演化
- 识别投资事件的时序关系（之前、之后、重叠）
- 支持时间范围查询和筛选

### 5.2 多模型融合推理

采用Stacking融合策略，结合多个模型的优势：
- PL-Marker：擅长实体标记关系抽取
- SPN：擅长集合预测关系抽取
- 加权融合：提高整体准确率

### 5.3 神经时序推理

实现MTTR（Multi-Task Temporal Reasoning）模型：
- 双任务训练：关系分类+时序关系分类
- BERT编码器：捕获深层语义信息
- 与规则推理融合：提高推理准确性

### 5.4 实体对齐

实现FTRLIM（Follow-The-Regularized-Leader for Entity Matching）模型：
- 在线学习：支持增量更新
- 多特征融合：编辑距离、Jaccard相似度、N-gram相似度、类型匹配
- 中英文实体对齐：支持别名识别

## 6. 总结与展望

### 6.1 工作总结

本项目成功设计并实现了融合时序特征的投融资风险知识图谱可视化系统，主要完成以下工作：

1. **完成了系统架构设计：** 采用四层架构，实现了前后端分离、ML服务独立部署
2. **实现了NLP服务：** 包括实体识别、关系抽取、时序推理等核心功能
3. **构建了知识图谱：** 支持388家企业、87家投资机构、4388条投融资事件
4. **实现了可视化分析：** 包括图谱展示、时序分析、风险传导等功能
5. **完成了模型训练：** 训练了PL-Marker、SPN、MTTR、FTRLIM四个模型
6. **通过了完整测试：** 73个测试断言全部通过，系统稳定可靠

### 6.2 创新点总结

1. **时序四元组模型：** 将时间信息融入知识图谱，支持时序分析
2. **多模型融合推理：** 结合多个模型优势，提高抽取准确率
3. **神经时序推理：** 实现MTTR模型，支持复杂的时序关系推理
4. **实体对齐：** 支持中英文实体对齐，提高知识融合能力

### 6.3 不足与改进

#### 存在不足：

1. **数据规模有限：** 当前数据量相对较小，可能影响模型泛化能力
2. **模型轻量化：** BERT模型较大，推理速度有待优化
3. **实时性：** 实体对齐和关系抽取的实时性有待提高
4. **多语言支持：** 目前主要支持中英文，其他语言支持有限

#### 改进方向：

1. **扩展数据规模：** 采集更多投融资数据，提高模型泛化能力
2. **模型压缩：** 采用知识蒸馏、量化等技术，提高推理速度
3. **增量学习：** 实现增量学习机制，支持模型在线更新
4. **多模态融合：** 融合文本、图像、表格等多模态信息

### 6.4 未来展望

1. **智能投研助手：** 基于知识图谱构建智能投研助手，提供投资建议
2. **风险预警系统：** 实现投融资风险实时预警功能
3. **跨领域应用：** 将技术扩展到其他金融领域（如信贷、保险）
4. **产业生态构建：** 开放API接口，构建投融资数据生态
